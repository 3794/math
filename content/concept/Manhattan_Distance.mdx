# Manhattan Distance

기하학중의 하나인 Taxicab geometry(택시 기하학) 에서 거리를 부를때 주로 Manhattan distance라고 부른다.

🎯 Computer Science에서 Manhattan Distance의 장점

1️⃣ 계산 효율성 (O(1) 연산)

- Manhattan: |Δx| + |Δy| ← 덧셈만 필요
- Euclidean: √(Δx² + Δy²) ← 제곱근 연산 필요

**영향:**

- 제곱근은 CPU 집약적 연산 (부동소수점 계산, 반복 알고리즘)
- Manhattan은 정수 연산만으로 처리 가능
- 대규모 거리 계산 (n×m 그래프, 클러스터링): O(n) vs O(n√n) 차이

2️⃣ 정수 좌표 최적화

고정소수점 또는 정수 격자에서:

- 게임 엔진 (타일 맵, 턴 기반 이동)
- 로봇 경로 계획 (격자 환경)
- 네트워크 라우팅 (라우터 배치)

Manhattan Distance는 부동소수점 오차 없이 정확한 결과 제공 ✓

3️⃣ 현실 세계 제약 반영

**격자 기반 이동 환경**에서 Euclidean 거리는 부정확함:

예: 도시 블록 이동

- 직선거리: √(3² + 4²) = 5 블록
- 실제 이동: |3| + |4| = 7 블록

더 나은 예: 로봇 팔(arm) 제어

- 관절(joint)은 독립적 축으로 작동
- 각 축의 거리를 합산하는 것이 현실적 ✓

4️⃣ 고차원 데이터에서의 확장성

d 차원 공간에서:

- Manhattan: O(d) 계산
- Euclidean: O(d) 계산 + √(d-term 합) = 더 느림

**머신러닝 응용:**

- 텍스트 유사도 (L1 거리)
- 이상 탐지 (Lasso 정칙화)
- 최근접 이웃 탐색 (KNN에서 L1 메트릭)

💡 왜 "맨해튼"인가?

뉴욕 맨해튼의 격자형 도로를 생각하면:

- 대각선 이동 불가능 (직각으로만 이동)
- X 방향 + Y 방향으로 가는 거리만 센다

목표점까지 거리 계산  
 ↓ ← (왼쪽 3블록)  
 ↓ ↓ (아래 2블록)  
 = 3 + 2 = 5블록 (Manhattan)

직선거리는 √(3²+2²) ≈ 3.6블록 (Euclidean)
