# Taxicab Geometry (택시 기하학)

**택시 기하학(Taxicab Geometry)** 은 직교좌표계에서 거리를 직선 경로만으로 측정하는 기하학이다. 뉴욕 맨해튼의 격자형 도로처럼 대각선 이동이 불가능한 환경에서 유래했으며, [Manhattan Distance](/concept/Manhattan_Distance)를 사용한다.

## 개념

### 유클리드 기하학 vs 택시 기하학

**유클리드 거리** (직선거리)
$$d_E = \sqrt{(x_2-x_1)^2 + (y_2-y_1)^2}$$

**택시 거리** (직각 경로만 가능)
$$d_T = |x_2-x_1| + |y_2-y_1|$$

### 예시: 맨해튼의 도시 블록

점 A(0, 0)에서 점 B(3, 2)로 가는 거리:

- 직선거리: $\sqrt{3^2 + 2^2} = \sqrt{13} ≈ 3.6$ 블록
- 택시 거리: $|3| + |2| = 5$ 블록

## 택시 기하학의 특징

### 1. 원의 모양이 다르다

**유클리드 기하학의 원:** 모두 같은 형태의 원

**택시 기하학의 원:** 마름모 모양

```
    *
   * *
  *   *
 * * * *
  *   *
   * *
    *
```

중심에서 거리 2인 모든 점들을 연결하면 마름모가 된다.

### 2. 피타고라스 정리가 다르다

택시 기하학에서는 직각삼각형이 다르게 정의된다.

**직선 거리에서:**
$$a^2 + b^2 = c^2$$

**택시 거리에서:**
적용되지 않음. 대신 직각 경로의 합으로 표현됨.

### 3. 최단 경로가 많다

유클리드 기하학에서는 두 점 사이의 최단 경로가 직선 하나이지만, 택시 기하학에서는 여러 경로가 같은 최단 거리를 가진다.

예: (0,0)에서 (2,2)로 가는 최단 경로

- 오른쪽 → 오른쪽 → 위 → 위
- 오른쪽 → 위 → 오른쪽 → 위
- 위 → 오른쪽 → 위 → 오른쪽
- 위 → 위 → 오른쪽 → 오른쪽

모두 거리가 4로 같다.

## 실제 응용

### 1. 도시 계획

- 도시 블록 사이의 실제 이동 거리 계산
- 응급차, 택시 배차 시스템
- 배송 경로 최적화

### 2. 로봇 공학

- 격자 환경에서의 경로 계획
- 체스나 체커 같은 보드 게임의 최단 경로

### 3. 컴퓨터 과학

- 이미지 처리에서 픽셀 간 거리 계산
- 네트워크 라우팅 (라우터는 격자 네트워크에서 동작)
- 데이터베이스 인덱싱

### 4. 머신 러닝

- L1 정칙화 (Lasso)의 기하학적 의미
- 이상 탐지 알고리즘

## 택시 기하학의 메트릭 성질

택시 거리는 다음을 만족하는 유효한 거리 메트릭이다:

1. **비음성**: $d(A,B) \geq 0$ (항상 0 이상)
2. **동일성**: $d(A,B) = 0$ ⟺ $A = B$
3. **대칭성**: $d(A,B) = d(B,A)$
4. **삼각부등식**: $d(A,C) \leq d(A,B) + d(B,C)$

이 성질들 때문에 택시 거리는 수학적으로 정당한 거리 함수로 사용된다.

## 일반화

택시 기하학은 3차원 이상에서도 정의된다:

**3차원 택시 거리:**
$$d = |x_2-x_1| + |y_2-y_1| + |z_2-z_1|$$

**n차원 택시 거리:**
$$d = \sum_{i=1}^{n} |x_i - y_i|$$

## 역사

택시 기하학이라는 이름은 뉴욕의 맨해튼 지역에서 유래했지만, 수학적으로는 **L1 메트릭** 또는 **Manhattan metric**이라고도 불린다. 이는 유클리드의 L2 메트릭과 구분되는 중요한 메트릭이다.

## 정리

택시 기하학은:

- ✓ 격자 기반 환경에서 실제 거리를 정확히 표현
- ✓ 유클리드 기하학보다 계산이 간단
- ✓ 현대의 도시, 컴퓨터, 네트워크 시대에 매우 실용적
- ✓ 높은 차원에서도 쉽게 적용 가능
